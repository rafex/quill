# PLANNING.md

Workflow para convertir una spec en plan ejecutable.

## Pasos

1. Leer problema, objetivo, alcance y criterios de aceptacion
   de la spec en `../specs/<iniciativa>/SPEC.md`.
2. Identificar dependencias, riesgos y restricciones.
3. Dividir el trabajo en tareas que produzcan evidencia observable.
4. Asignar orden, dependencias y criterio de cierre a cada tarea.
5. Registrar el resultado en `../tasks/<iniciativa>/TASKS.md`.
